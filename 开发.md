# 搜打撤物资系统设计参考：格子·拖拽·双击

> 本文档整理自《三角洲行动》游戏机制分析，结合当前项目（Vue3+TS）的现有实现，供开发参考。

## 一、格子系统

### 1.1 物品占格规则（三角洲实现）

物品占用格子数量从 1×1 到 3×4（12格）不等，**同样的格数还有横向/纵向之分**，这直接影响背包空间利用率。

| 物品格数 | 形状 | 典型物品 | 示例 |
|---------|------|---------|------|
| **1格** | 1×1 | 非洲之心、钻石鱼子酱、吴彦祖之镜 | 任意空格可放 |
| **2格** | 横向 2×1 | 显卡、军用信息控制终端、棘龙爪化石 | 横向长条 |
| **2格** | 纵向 2×1 | 金条、高级咖啡豆、香槟 | 纵向长条 |
| **4格** | 正方形 2×2 | 军用电台、摄影机、军用无人机、呼吸机 | 方形中件 |
| **4格** | 纵向 1×4 | 滑膛枪展品 | 纵向长条 |
| **6格** | 横向 2×3 | 笔记本电脑、飞行记录仪、军用炮弹 | 横向扁块 |
| **6格** | 纵向 2×3 | 自动体外除颤器、医疗机械人 | 纵向竖块 |
| **9格** | 正方形 3×3 | 复苏呼吸机、ECMO、主战坦克模型 | 大件物品 |
| **12格** | 横向 3×4 | 高速磁盘阵列、动力电池组 | 超大件 |
| **12格** | 纵向 3×4 | 刀片服务器 | 超大竖件 |

> **"大红"定义**：6格及以上 + 红色品质，是游戏内最高价值物品。

### 1.2 多格物品的嵌套规则

这是格子系统的核心深度设计：

```
多格背包可装比自身小的物品
├── 6格包 → 可装 4格、3格、2格、1格
├── 4格包 → 可装 2格、1格
├── 4格包 → ❌ 不能装 3格（中间缺一格，放不下）
└── 9格包 → 可装 6格、4格、3格、2格、1格
```

> **关键规则**：只能装"能完整放下"的物品，缺一格都放不下。这迫使玩家做空间取舍。

### 1.3 容器的"格式"概念

物资箱（容器）本身也有格子布局概念（长×宽），决定是否出大红：

| 容器格式 | 是否出大红 | 说明 |
|---------|-----------|------|
| **3×3 / 3×4 / 4×3** | ✅ 必出 | 高价值大红区 |
| **1×3 / 2×4 / 3×1** | ❌ 必空 | 小格式多为空盒 |
| 武器箱 3×3 | 碳纤维板 | 固定产出 |
| 武器箱 3×2 | 燃料电池 | 固定产出 |

## 二、拖拽操作

### 2.1 三角洲的拖拽交互

| 操作 | 行为 |
|------|------|
| 拖拽物品到空格 | 物品移动到该格 |
| 拖拽物品到有物品的格 | **两物品交换位置** |
| 拖拽物品到无效位置 | 物品返回原位 |
| 拖拽物品到胸挂/口袋专用格 | 仅允许对应类型物品进入 |
| 拖拽到容器外 | 丢弃物品（某些游戏） |

### 2.2 当前项目的拖拽实现

使用 HTML5 原生 Drag and Drop API：

**BagCell（背包格子）**：
```vue
<!-- @/components/game/BagCell.vue -->
<div
  class="bag-cell"
  draggable="true"
  @dragstart="onDragStart"
  @dblclick="onDblClick"
>
```
- `@dragstart`：拖拽开始时通知父组件，携带 `instanceId`
- 未搜索的物资箱物品禁止拖拽（`e.preventDefault()`）

**LootUI（物资箱面板）**：
```vue
<!-- @/components/game/LootUI.vue -->
<div
  class="cell"
  @drop.prevent="onBagDrop($event, r-1, c-1)"
  @dragover.prevent
>
```

**Store 层拖拽处理**：
```typescript
// @/stores/gameStore.ts

// 物资箱→背包拖拽
function placeItemInBag(instanceId: number, row: number, col: number): boolean {
  if (bagGrid.value[row][col] !== null) return false  // 目标格有物则拒绝
  // ...
  bagItems.value.push({ instanceId, itemId: lootItem.itemId, row, col })
  box.items.splice(idx, 1)
  return true
}

// 背包内移动
function moveBagItem(instanceId: number, toRow: number, toCol: number): boolean {
  if (bagGrid.value[toRow][toCol] !== null) return false  // 目标格有物则拒绝
  const item = bagItems.value.find(i => i.instanceId === instanceId)
  item.row = toRow
  item.col = toCol
  return true
}
```

### 2.3 当前实现的局限性

| 问题 | 说明 |
|------|------|
| 不支持物品交换 | `placeItemInBag` 在目标格非空时直接 return false |
| 不支持跨背包拖拽 | 只能从物资箱拖到背包，或背包内移动 |
| 不支持放回物资箱 | 需要双击操作 |
| 无多格物品碰撞检测 | 当前是单格，无需检测 |

### 2.4 改进建议

```typescript
// 改进：支持物品交换
function placeItemInBag(instanceId: number, row: number, col: number): boolean {
  const existingItem = bagGrid.value[row][col]
  if (existingItem) {
    // 交换：把现有物品放回物资箱
    returnItemToBox(existingItem.instanceId)
  }
  // ... 继续放置逻辑
}

// 改进：支持物资箱格子的拖拽放置
// 物资箱格子也需要 @drop/@dragover 事件
```

## 三、双击操作

### 3.1 三角洲的双击交互

| 场景 | 操作 | 行为 |
|------|------|------|
| 物资箱内双击物品 | **双击** | 物品直接进入背包（或最近有空位的格） |
| 背包内双击物品 | **双击** | 物品放回原物资箱 |
| 敌人盒子内双击物品 | **双击** | 物品进入背包 |
| 舔包时双击 | **双击** | 快速拾取，优先放背包主格 |

**关键设计**：
- 双击是**最快速的拾取方式**，无需精确拖拽
- 双击**自动找空位**，不需要玩家手动指定格子
- 未揭示的物品（搜索动画进行中）**禁止双击**

### 3.2 当前项目的双击实现

**BagCell（背包格子）**：
```typescript
function onDblClick() { emit('dblclick-item', props.item.instanceId) }
```

**LootCell（物资箱格子）**：
```typescript
function onDblClick() {
  if (props.item.searched) emit('dblclick-item', props.item.instanceId)
  // 未搜索的物品：双击无效，不 emit
}
```

**LootUI（面板层）**：
```typescript
// 物资箱双击 → 拾取到背包
function onLootDblClick(instanceId: number) {
  store.pickItemToBag(instanceId)
}

// 背包双击 → 放回物资箱
function onBagDblClick(instanceId: number) {
  store.returnItemToBox(instanceId)
}
```

**Store 层（核心逻辑）**：
```typescript
// 物资箱→背包：自动找空位
function pickItemToBag(instanceId: number): boolean {
  const placement = findBagPlacement()  // 自动找第一个空位
  if (!placement) return false  // 背包满则失败
  bagItems.value.push({ instanceId, itemId: lootItem.itemId, row: placement.row, col: placement.col })
  box.items.splice(idx, 1)
  return true
}

// 背包→物资箱：自动找空位
function returnItemToBox(instanceId: number): boolean {
  // 找物资箱第一个空位
  for (let c = 0; c < 6; c++) {
    if (!occupied.has(`0,${c}`)) { col = c; break }
  }
  box.items.push({ instanceId, itemId: bagItem.itemId, searched: true, row, col })
  bagItems.value.splice(bagIdx, 1)
  return true
}
```

## 四、物资流转总览

### 4.1 完整路径图

```
[地图物资箱]（局内）
    ↓ 双击 / 拖拽
[背包主格子 / 胸挂 / 口袋]（局内）
    ↓ 撤离成功
[仓库]（局外）

[仓库]（局外）
    ↓ 进局前配置
[背包 / 安全箱]（局内）

死亡/超时：
  背包内容 → ❌ 全丢
  安全箱内容 → ✅ 保留
  仓库内容 → ✅ 永远安全
```

### 4.2 五类空间的职能划分

| 空间 | 局内/局外 | 死亡保护 | 快捷键访问 | 主要用途 |
|------|---------|---------|-----------|---------|
| **物资箱** | 局内 | — | — | 物品来源 |
| **背包主格子** | 局内 | ❌ | Tab 打开 | 战利品存放 |
| **胸挂** | 局内 | ❌ | R 键直接 | 弹药快捷 |
| **口袋** | 局内 | ❌ | 5 键直接 | 药品快捷 |
| **安全箱** | 局内 | ✅ | Tab 打开 | 高价值保底 |
| **仓库** | 局外 | ✅ | 局外菜单 | 长期存储 |

| 空间 | 实现状态 | 说明 |
|------|---------|------|
| 物资箱 | ✅ 已实现 | LootUI 左侧面板，搜索动画 |
| 背包 | ✅ 已实现 | 10×6 格子，BagCell |
